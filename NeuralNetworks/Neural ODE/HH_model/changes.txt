================================================================================
HH Neural ODE - Refactoring Changelog
================================================================================

OVERVIEW
--------
Resolved module separation issues where loss/optimization functions were
duplicated or broken across files. All loss and optimization logic is now
centralized in physics_loss.py. Plotting functions extracted to visualization.py.
Fixed critical bugs that would have caused runtime errors.


================================================================================
FILE: physics_loss.py  (MAJOR CHANGES)
================================================================================

1. BUG FIX: physics_residual() — Fixed broken function signature and body
   - OLD (line 56): physics_residual(model, t_points, y_points, I_ext_fn, hh)
     Called hh.derivatives() which DOES NOT EXIST on HodgkinHuxley class.
     Expected I_ext_fn as a callable, but callers passed arrays.
   - NEW: physics_residual(model, hh, V_samples, t_samples, I_ext_model, I_ext_hh)
     Now accepts arrays directly (matching how it's actually called).
     Builds full 4D state [V, m_inf, h_inf, n_inf] from voltage using
     HodgkinHuxley steady-state functions.
     Uses jax.vmap(hh.__call__) instead of non-existent hh.derivatives().

2. BUG FIX: adversarial_physics_loss() — Fixed call to physics_residual
   - OLD (line 99): physics_residual(model, hh, V_samples, t_samples, ...)
     Argument order mismatched the OLD function signature (model, t_points,
     y_points, I_ext_fn, hh). Would have crashed at runtime.
   - NEW: Signature now matches — both use (model, hh, V_samples, t_samples,
     I_ext_model, I_ext_hh).

3. REMOVED: total_loss_fn() (was lines 122-167)
   - Was a placeholder with data_loss hardcoded to 0.0 and a TODO comment.
   - Never called by any file. Replaced by shooting_combined_loss which
     actually computes data loss via segment integration.

4. REMOVED: minimax_step() (was lines 170-225)
   - Only did physics-only minimax without shooting integration.
   - Was never called — train.py used its own make_shooting_train_step()
     which wraps the full shooting_combined_loss.
   - Replaced by make_shooting_train_step() moved from train.py.

5. ADDED: shooting_data_loss() — moved from multiple_shooting.py (was line 142)
   MSE loss on voltage predictions across all segments.

6. ADDED: continuity_loss() — moved from multiple_shooting.py (was line 121)
   Penalizes discontinuities at segment boundaries.

7. ADDED: shooting_gating_penalty() — moved from multiple_shooting.py (was line 157)
   Penalizes gating variables outside [0, 1].
   BUG FIX: Changed jnp.max() to jnp.mean(). The old version used jnp.max
   which only backpropagates gradients through the single worst violation
   point, providing zero gradient signal for all other violations. Using
   jnp.mean provides smooth gradients across all violations, improving
   optimization stability.

8. ADDED: shooting_combined_loss() — moved from multiple_shooting.py (was line 178)
   Full multiple-shooting loss combining data, continuity, physics, and
   gating penalty terms.

9. ADDED: make_shooting_train_step() — moved from train.py (was line 262)
   Factory creating JIT-compiled minimax training step. Uses closure pattern
   to capture static data for BacksolveAdjoint compatibility.

10. UPDATED: Module imports
    - Added: from multiple_shooting import integrate_all_segments
    - Removed: import optax (no longer needed at module level)

11. UPDATED: Quick test (__main__ block)
    - Simplified to only test physics_residual and adversarial_physics_loss.
    - Removed minimax_step test (function no longer exists).

   [was in multiple_shooting.py line 170, now in physics_loss.py]


================================================================================
OPTIMIZATION ISSUES IDENTIFIED
================================================================================

1. jnp.max in gating penalty (FIXED above): Using max over all gating
   violations means gradients only flow through the single worst violation.
   All other out-of-bound gating variables receive zero gradient. Changed
   to jnp.mean for smooth, distributed gradient signal.

2. Duplicate forward pass for weight gradients: make_shooting_train_step
   computes shooting_combined_loss twice — once for model gradients and once
   for weight gradients. Since the loss depends on both model and
   loss_weights, a single forward pass with jax.grad w.r.t. both could
   halve the integration cost. However, eqx.filter_value_and_grad only
   differentiates the first argument, and the closure/JIT pattern makes
   this refactor non-trivial. Noted for future optimization.

3. minimax_step() and total_loss_fn() in physics_loss.py were dead code —
   never called by any module. The actual training used
   make_shooting_train_step() in train.py which reimplemented the minimax
   logic around shooting_combined_loss. Now consolidated.
